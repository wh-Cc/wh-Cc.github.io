代码验证和自动格式化是现代前端开发中的重要实践，它们帮助保持代码质量和一致性。

以下是如何在项目中实现代码验证和自动格式化的步骤：

1. 代码验证 (Linting) ​

使用 ESLint 作为代码验证工具：

  • 安装 ESLint:

    sh
    npm install eslint --save-dev

  • 初始化 ESLint:

    sh
    npx eslint --init

    这个命令会引导你通过一系列问题来创建 .eslintrc 配置文件。

  • 配置 ESLint: 在 .eslintrc 文件中，你可以配置规则、环境、扩展等。

  • 运行 ESLint:

    sh
    npx eslint your-file.js

    或者添加到 package.json 的脚本中：

    json
    {
	"scripts": {
		"lint": "eslint src/"
	}
}

2. 自动格式化 ​

使用 Prettier 作为代码格式化工具：

  • 安装 Prettier:

    sh
    npm install prettier eslint-plugin-prettier eslint-config-prettier --save-dev

  • 集成 Prettier 到 ESLint: 在 .eslintrc 配置文件中添加 Prettier 插件和配置：

    json
    {
	"plugins": ["prettier"],
	"extends": ["eslint-config-prettier"],
	"rules": {
		"prettier/prettier": "error"
	}
}

  • 运行 Prettier:

    sh
    npx prettier --write your-file.js

    或者使用 ESLint 一起运行：

    sh
    npx eslint --fix your-file.js

    --fix 选项会让 ESLint 尝试自动修复可修复的问题。

3. 配置编辑器 ​

大多数现代编辑器（如 VS Code）都支持 ESLint 和 Prettier：

  • VS Code 设置: 在 VS Code 中，你可以安装 ESLint 和 Prettier 插件，并在设置中启用它们：
    json
    {
	"editor.codeActionsOnSave": {
		"source.fixAll.eslint": true
	},
	"eslint.validate": ["javascript", "typescript", "vue"],
	"prettier.enable": true,
	"prettier.useEditorConfig": true
}

4. 保存时格式化 (On-save Formatting) ​

配置编辑器在保存文件时自动格式化代码：

  • VS Code: 在 settings.json 中设置 "editor.formatOnSave": true。

5. 钩子 (Hooks) ​

使用 Husky 和 lint-staged 来在提交前自动运行代码验证和格式化：

  • 安装 Husky 和 lint-staged:

    sh
    npm install husky lint-staged --save-dev

  • 配置 Husky: 在 package.json 中配置 Husky 钩子：

    json
    {
	"husky": {
		"hooks": {
			"pre-commit": "lint-staged"
		}
	}
}

  • 配置 lint-staged:

    json
    {
	"lint-staged": {
		"*.{js,ts,vue}": ["eslint --fix", "prettier --write"]
	}
}

通过这些步骤，你可以确保代码在提交到仓库之前满足质量标准，并且保持一致的代码风格。这有助于减少代码审查中的样式问题，让团队成员专注于代码逻辑和功能实现。

在create-vue 中配置 ​

create-vue 中自带了 eslint 和 prettier

eslint 配置

javascript
// 指定该文件应在 Node.js 环境中执行
/* eslint-env node */

// 引入 Rush Stack 的 ESLint 补丁，用于现代模块解析
require("@rushstack/eslint-patch/modern-module-resolution");

// 导出 ESLint 配置对象
module.exports = {
	// 标记为根配置，意味着这是项目中最顶层的配置文件
	root: true,

	// 扩展其他 ESLint 配置
	extends: [
		// 为 Vue 3 项目提供基本的 ESLint 插件配置
		"plugin:vue/vue3-essential",
		// 包含 ESLint 默认的推荐规则
		"eslint:recommended",
		// Vue 项目中使用 TypeScript 的 ESLint 配置
		"@vue/eslint-config-typescript",
		// 与 Prettier 配合使用的 ESLint 配置，跳过格式化检查
		"@vue/eslint-config-prettier/skip-formatting",
	],

	// 覆盖特定文件或目录的 ESLint 配置
	overrides: [
		{
			// 指定覆盖的文件匹配模式，这里针对 Cypress 测试文件
			files: [
				"cypress/e2e/**/*.{cy,spec}.{js,ts,jsx,tsx}",
				"cypress/support/**/*.{js,ts,jsx,tsx}",
			],

			// 为匹配的文件扩展特定的 ESLint 配置
			extends: [
				// 应用 Cypress 推荐的 ESLint 插件配置
				"plugin:cypress/recommended",
			],
		},
	],

	// 设置解析器选项，指定 ECMAScript 版本为最新
	parserOptions: {
		ecmaVersion: "latest",
	},
};

prettier 配置

javascript

    //指定了配置文件的模式
    "$schema": "https://json.schemastore.org/prettierrc&#34

    //设置 Prettier 不在语句末尾添加分号
    "semi": false

    //设置每个制表符（tab）等于两个空格
    "tabWidth": 2

    //让 Prettier 使用单引号代替双引号
    "singleQuote": true

    //设置每行代码的最大字符数，超过这个数值时，Prettier 会尝试换行
    "printWidth": 100

    //设置 Prettier 不在行末添加逗号
    "trailingComma": "none"

需手动添加 stylelint

sh
npm install stylelint stylelint-config-standard --save-dev

然后，创建一个 .stylelintrc 配置文件：

json
{
  "extends": "stylelint-config-standard"
}

你可以在 package.json 中添加一个 stylelint 脚本：

json
{
  "scripts": {
    "lint:style": "stylelint '**/*.vue'"
  }
}
